# Retail Sales Analysis Using SQL

## Project Overview

This project analyzes a retail sales dataset using SQL to uncover customer purchasing behavior, sales trends, category performance, and customer insights. The goal is to demonstrate practical SQL skills including filtering, aggregation, grouping, window functions, date/time analysis, and business reporting.

## Dataset Description

The dataset contains retail transaction records with the following attributes:

| Column Name     | Description                   |
| --------------- | ----------------------------- |
| transactions_id | Unique transaction identifier |
| sale_date       | Date of purchase              |
| sale_time       | Time of purchase              |
| customer_id     | Unique customer identifier    |
| gender          | Customer gender               |
| age             | Customer age                  |
| category        | Product category              |
| quantity        | Number of items purchased     |
| price_per_unit  | Price per unit                |
| cogs            | Cost of goods sold            |
| total_sale      | Total transaction value       |

## Project Objectives

The project aims to answer key business questions such as:

* Which product categories generate the highest sales?
* What are the purchasing patterns of customers?
* Which months perform best in terms of sales?
* Who are the top customers?
* How does customer behavior vary across different shifts of the day?

## SQL Analysis Performed

### Q1. Sales on a Specific Date

Retrieved all sales transactions made on **2022-11-05**.

### Q2. Clothing Sales Analysis

Retrieved transactions where:

* Category = Clothing
* Quantity sold > 4
* Purchase month = November 2022

### Q3. Total Sales by Category

Calculated total revenue generated by each product category.

### Q4. Average Age of Beauty Customers

Computed the average age of customers purchasing products from the Beauty category.

### Q5. High-Value Transactions

Identified transactions where total sales exceeded 1000.

### Q6. Transactions by Gender and Category

Calculated the number of transactions made by each gender within each category.

### Q7. Monthly Average Sales

Calculated average sales value for each month.

### Q8. Best Selling Month Each Year

Identified the highest-performing month in each year using SQL window functions.

### Q9. Top 5 Customers

Found the top 5 customers based on total spending.

### Q10. Unique Customers by Category

Calculated the number of unique customers purchasing from each category.

### Q11. Sales Shift Analysis

Segmented transactions into:

* Morning (< 12:00)
* Afternoon (12:00 – 16:59)
* Evening (>= 17:00)

and calculated the number of orders in each shift.
