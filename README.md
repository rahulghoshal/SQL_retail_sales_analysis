# Retail Sales Analysis Using SQL

## Project Overview

This project focuses on analyzing a retail sales dataset using SQL to uncover customer behavior, sales trends, and business insights. The analysis was performed using MySQL and covers various aspects such as sales performance, customer demographics, product categories, and transaction patterns.

## Objectives

The primary objectives of this project are:

* Analyze sales transactions and customer purchasing behavior.
* Identify top-performing product categories.
* Discover sales trends across different months and years.
* Segment transactions based on time of day.
* Identify high-value customers.
* Practice SQL concepts including filtering, aggregation, grouping, window functions, and conditional logic.

## Dataset Information

The dataset contains retail transaction records with the following attributes:

| Column Name     | Description                   |
| --------------- | ----------------------------- |
| transactions_id | Unique transaction identifier |
| sale_date       | Date of transaction           |
| sale_time       | Time of transaction           |
| customer_id     | Unique customer identifier    |
| gender          | Customer gender               |
| age             | Customer age                  |
| category        | Product category              |
| quantity        | Quantity purchased            |
| price_per_unit  | Price per item                |
| cogs            | Cost of goods sold            |
| total_sale      | Total transaction value       |

## Business Questions & SQL Analysis

### 1. Sales on a Specific Date

Retrieve all sales made on 18-12-2022.

### 2. Clothing Transactions Analysis

Retrieve all Clothing category transactions where quantity sold was less than 4 during November 2022.

### 3. Category-wise Revenue Analysis

Calculate total sales generated by each product category.

### 4. Customer Demographics

Find the average age of customers who purchased products from the Beauty category.

### 5. High-Value Transactions

Identify transactions with total sales greater than 1000.

### 6. Gender and Category Analysis

Determine the number of transactions made by each gender across different product categories.

### 7. Monthly Sales Performance

Calculate the average sales value for each month.

### 8. Best Selling Month Analysis

Identify the best-selling month in each year based on average sales.

### 9. Top Customers

Find the top 5 customers based on total sales generated.

### 10. Customer Segmentation by Category

Determine the number of unique customers purchasing from each category.

### 11. Shift-Based Order Analysis

Segment orders into operational shifts:

* Morning: Before 12 PM
* Afternoon: 12 PM to 5 PM
* Evening: After 5 PM

Analyze the number of orders generated during each shift.
