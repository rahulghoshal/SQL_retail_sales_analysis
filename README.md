# Retail Sales Analysis using SQL & Power BI

## Project Overview

This project focuses on analyzing retail sales data using **MySQL** and visualizing key business insights through an interactive **Power BI Dashboard**.

The objective is to explore customer purchasing behavior, identify sales trends, analyze category performance, and generate actionable business insights through SQL queries and data visualization.

---

## Dataset Information

The dataset contains retail transaction records with the following fields:

| Column Name     | Description                   |
| --------------- | ----------------------------- |
| transactions_id | Unique transaction identifier |
| sale_date       | Date of transaction           |
| sale_time       | Time of transaction           |
| customer_id     | Unique customer identifier    |
| gender          | Customer gender               |
| age             | Customer age                  |
| category        | Product category              |
| quantity        | Quantity purchased            |
| price_per_unit  | Price per item                |
| cogs            | Cost of goods sold            |
| total_sale      | Total transaction value       |

### Dataset Size

* **Total Records:** 2000
* **Cleaned Records Used for Analysis:** 1987

## SQL Business Problems Solved

### 1. Sales on a Specific Date

Retrieved all sales transactions made on **18-12-2022**.

### 2. Clothing Category Analysis

Retrieved all Clothing transactions where:

* Quantity sold < 4
* Month = November 2022

### 3. Category-wise Revenue Analysis

Calculated total sales revenue generated by each product category.

### 4. Customer Demographic Analysis

Calculated the average age of customers purchasing products from the Beauty category.

### 5. High-Value Transactions

Retrieved transactions where total sales exceeded 1000.

### 6. Gender-wise Category Analysis

Calculated the number of transactions made by each gender across product categories.

### 7. Monthly Sales Analysis

Calculated average sales for each month.

### 8. Best Selling Month Analysis

Used SQL Window Functions (`RANK()`) to identify the best-performing month in each year.

### 9. Top Customer Analysis

Identified the Top 5 customers based on total revenue generated.

### 10. Unique Customer Analysis

Calculated the number of unique customers purchasing from each category.

### 11. Shift-wise Sales Analysis

Created customer purchase shifts using SQL CASE statements:

| Shift     | Time Range   |
| --------- | ------------ |
| Morning   | Before 12 PM |
| Afternoon | 12 PM – 5 PM |
| Evening   | After 5 PM   |

Calculated the total number of orders in each shift.

## Key Insights

* Electronics generated the highest overall sales revenue.
* Clothing and Electronics categories contributed significantly to total sales.
* Certain months outperformed others, indicating seasonal purchasing behavior.
* The 46–55 age group generated the highest sales.
* Customer purchasing patterns varied across different shifts.
